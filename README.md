# 1. 제목(Headers):
## 1.큰 제목(Heading 1): 가장 큰 제목을 만들 때는 # 기호를 사용합니다. 이 때 # 기호 뒤에 한 칸을 띄어서 제목을 작성합니다. 예를 들어, 다음과 같이 작성하면 됩니다: # 아 이게 야스지
## 2.작은 제목(Heading 2): 더 작은 제목을 만들 때는 ## 기호를 사용합니다. 예를 들어, 다음과 같이 작성하면 됩니다: ## 아 이게 야스지
## 3.중간 크기 제목(Heading 3): 더 작은 제목을 만들 때는 ### 기호를 사용합니다. 예를 들어, 다음과 같이 작성하면 됩니다: ### 아 이게 야스지
## 4.제목 수준이 더 낮은 경우(Heading 4 이상): 필요한 경우 ####, #####, ######과 같이 더 많은 # 기호를 사용하여 제목의 수준을 낮출 수 있습니다. 예를 들어, 다음과 같이 작성하면 됩니다:#### 아 이게 야스지

# 2. 볼드체와 이탤릭체(Bold and Italic):
## 1. 볼드체(Bold): 텍스트를 볼드체로 표시하려면 텍스트를 ** 또는 __ 로 감싸면 됩니다. 예를 들어, 다음과 같이 작성하면 됩니다: **아 아게 야스지** 또는 __아 이게 야스지__
## 2. 이탤릭체(Italic): 텍스트를 이탤릭체로 표시하려면 텍스트를 * 또는 _ 로 감싸면 됩니다. 예를 들어, 다음과 같이 작성하면 됩니다: *아 이게 야스지* 또는 _아 이게 야스지_

# 3. 리스트(Lists):
## 1. 순서가 있는 리스트(ordered list): 숫자와 점을 사용하여 순서가 있는 리스트를 작성합니다. 아래와 같이 번호와 리스트 항목을 입력하면 됩니다.
### 1. 항목 1
### 2. 항목 2
### 3. 항목 3
## 2. 순서가 없는 리스트(unordered list): 빈칸과 별표를 사용하여 순서가 없는 리스트를 작성할 수 있습니다. 별표(*)나 빈칸(-)을 사용하면 됩니다.
### * 항목 1
### * 항목 2
### * 항목 3
### - 항목 1
### - 항목 2
### - 항목 3
## 3. 중첩된 리스트(nested list): 중첩된 리스트를 작성할 수도 있습니다. 이 경우에는 탭을 사용하여 들여쓰기를 하면 됩니다.
### * 항목 1
###    * 서브항목 1
###    * 서브항목 2
### * 항목 2
### * 항목 3

# 4. 링크(Links):
## 1. 인라인 링크(Inline link): 텍스트를 클릭 가능한 링크로 만들려면 [링크 텍스트](링크 URL) 형식을 사용합니다. 예를 들어, 다음과 같이 작성하면 됩니다:
### [GitHub](https://github.com/)
## 2. 참조 링크(Reference link): 링크를 사용할 때 링크 URL을 한 번 정의하고 나중에 그 URL을 참조하여 사용할 수도 있습니다. 이 방식은 긴 URL을 한 번에 관리하기에 편리합니다. 먼저, 링크 URL을 정의합니다:
### [GitHub][github-url] 
### [github-url]: https://github.com/
## 그리고 텍스트에서 해당 링크를 참조하여 사용합니다:
### [GitHub][github-url]
## 3. 자동 링크(Automatic link): URL 주소를 입력하면 자동으로 링크로 변환됩니다. 예를 들어, 다음과 같이 작성하면 됩니다:
### https://github.com/

# 5. 이미지(Images): 
## 1. 인라인 이미지(Inline image): 이미지를 텍스트 내에 삽입하려면 ![대체 텍스트](이미지 URL) 형식을 사용합니다. 이미지 URL은 이미지 파일이 위치한 웹상의 경로입니다. 대체 텍스트는 이미지가 로드되지 못했을 때 대신 표시될 텍스트입니다. 예를 들어, 다음과 같이 작성하면 됩니다:
### ![GitHub 로고](https://github.githubassets.com/images/modules/logos_page/GitHub-Mark.png)
## 2. 참조 이미지(Reference image): 이미지를 사용할 때 이미지 URL을 한 번 정의하고 나중에 그 URL을 참조하여 사용할 수도 있습니다. 이 방식은 긴 URL을 한 번에 관리하기에 편리합니다. 먼저, 이미지 URL을 정의합니다:
### ![GitHub 로고][github-logo]
### [github-logo]: https://github.githubassets.com/images/modules/logos_page/GitHub-Mark.png
## 그리고 텍스트에서 해당 이미지를 참조하여 사용합니다:
### ![GitHub 로고][github-logo]

# 6. 인라인 코드(Inline code): 
## 인라인 코드는 ` (역따옴표)로 코드를 감싸면 됩니다. 예를 들어, print("Hello, world!") 와 같이 코드를 표현하고자 할 때는 다음과 같이 작성합니다: `print("Hello, world!")`

# 7. 코드 블록(Code blocks):
## 코드 블록은 세 개의 역따옴표 (```)를 사용하여 표현됩니다. 코드 블록을 시작하는 세 개의 역따옴표 뒤에 언어를 지정하여 해당 언어의 구문 강조를 적용할 수 있습니다. 언어를 지정하지 않으면 일반 텍스트로 간주됩니다. 예를 들어, 파이썬 코드 블록을 작성하려면 다음과 같이 작성합니다:
### ```python
### def hello():
### print("Hello, world!")

### hello()
### ```
## 이렇게 작성하면 다음과 같이 코드 블록이 표시됩니다:
### def hello():
###    print("Hello, world!")

### hello()


# 8. 인용문(Blockquotes): 
## 텍스트를 인용문으로 만들 때 사용됩니다. 인용문을 작성하려면 > 기호를 사용하면 됩니다. 예를 들어, 다음과 같이 작성하면 됩니다:
### > 이것은 인용문입니다.
## 인용문 안에 또 다른 인용문을 중첩하여 작성할 수도 있습니다. 이 경우에는 > 기호를 중첩하여 사용하면 됩니다. 예를 들어:
### > 이것은 첫 번째 인용문입니다.
# >
# > > 이것은 두 번째 인용문입니다.


# 9. 수평선(Horizontal rules): 
## 텍스트를 구분하거나 섹션을 나눌 때 사용됩니다. 수평선을 작성하려면 세 개 이상의 하이픈(-), 별표(*), 혹은 언더바(_)를 연속해서 입력하면 됩니다. 예를 들어, 다음과 같이 작성하면 됩니다:
### --- 또는 *** 또는 ___

# 10. 테이블(Tables):
## 테이블을 작성하려면 세로선(|)을 사용하여 열을 구분하고, 각 행은 새로운 줄에 작성합니다. 첫 번째 행은 테이블의 헤더로 사용되며, 두 번째 행은 각 열의 정렬 방식을 나타냅니다. 그 이후의 행은 데이터를 포함합니다. 예를 들어, 다음과 같이 작성하면 됩니다:
### | 이름   | 나이 | 성별 |
### |--------|-----|------|
### | 홍길동 | 30  | 남자 |
### | 이순신 | 40  | 남자 |
### | 김영희 | 25  | 여자 |
## 각 열의 길이가 다를 경우 테이블의 가독성을 높이기 위해 좌우로 정렬할 수 있습니다. 열을 정렬하려면 각 열의 헤더 바로 아래에 하이픈(-)을 사용하고, 콜론(:)을 사용하여 정렬 방식을 지정합니다. 예를 들어, 다음과 같이 작성하면 됩니다:
### | 이름   | 나이 | 성별   |
### |--------|:----:|------:|
### | 홍길동 | 30   | 남자 |
### | 이순신 | 40   | 남자 |
### | 김영희 | 25   | 여자 |
